Processo de GeraçãoMistura (Hashing): Compacta e normaliza os bytes de entropia imprevisíveis.
Derivação: Garante distribuição uniforme de bits, eliminando padrões.
Codificação: Converte os bits resultantes em letras, números e símbolos.

Possibilidades de conversão de entropia. 

Descarte imediato: Apague os bytes brutos da memória após gerar a senha.
Não reutilize: Se precisar de nova senha, colete nova entropia do sistema.
Evite determinismo simples: Nunca use operadores matemáticos básicos (como + ou *) diretamente nos bytes para escolher caracteres, pois isso destrói a aleatoriedade.

Para implementar essa lógica em Linguagem C, você deve combinar as escolhas do usuário em um único vetor de caracteres (o alfabeto disponível) e usar a entropia misturada com uma função de hash (como SHA-256) para selecionar os índices de forma segura.

Segurança de Memória: O uso do memset(senha, 0, tamanho) no final é crucial. Ele garante que a senha gerada não fique flutuando na memória RAM após o término da execução.

Mapeamento por Módulo (%): O operador % tam_alfabeto garante que o valor derivado da entropia caia exatamente em um índice válido do vetor de caracteres configurado pelo usuário.

Complexidade na Lógica de Sorteio: Como a tabela ASCII é fragmentada (os números estão em um bloco, maiúsculas em outro e os especiais estão espalhados), você não pode simplesmente fazer índice % tamanho. O código precisará rodar em um laço (while) sorteando bytes da entropia até achar um que se encaixe nas regras do usuário (técnica de rejeição), o que consome mais processamento.Complexidade nos Caracteres Especiais: A tabela ASCII possui caracteres de controle invisíveis (como Line Feed ou Tab) e símbolos problemáticos (como o espaço ou aspas simples/duplas) que podem quebrar sistemas ou arquivos de configuração se incluídos na senha. Filtrar isso via código ASCII puro exige condicionais (if) complexas.Manutenção Difícil: Ler números como if (c >= 33 && c <= 47) torna o código menos legível para humanos do que olhar para uma string "=!@#$"
Complexidade Oculta: A lógica com ASCII exige tratar cenários onde a entropia "acaba". Se o usuário escolher apenas números e o seu hash gerar muitos bytes correspondentes a letras, o código precisará gerar mais hash em loop para continuar sorteando (amostragem por rejeição). O Mecanismo: Entropia em Bits vs. Espaço de BuscaA força real de uma senha gerada aleatoriamente é medida em bits de entropia (\(H\)). A fórmula para calcular a segurança real de uma senha gerada aleatoriamente é:\(H=x\times \log _{2}(N)\)Onde:\(x\) = Tamanho da senha.\(N\) = Tamanho do alfabeto escolhido.Se o usuário escolher um alfabeto simples (\(N=10\), apenas números), cada caractere adicionado injeta aproximadamente 3.32 bits de entropia (\(\log_2(10)\)). Se ele escolher o alfabeto completo (\(N=95\)), cada caractere injeta 6.57 bits (\(\log_2(95)\)).Como o seu sistema já define um mínimo de 16 caracteres:No pior cenário (apenas números, \(10^{16}\)): A senha terá 53 bits de entropia.No melhor cenário (todos os caracteres, \(95^{16}\)): A senha terá 105 bits de entropia.Nota: Em criptografia, qualquer valor acima de 80 bits é considerado computacionalmente invulnerável contra ataques atuais.


Como aplicar a Aleatoriedade focada no Tamanho (Sem o viés de tipos)Se o foco muda de "garantir que todos os tipos apareçam" para "maximizar a aleatoriedade pura ao longo do tamanho requisitado", o seu algoritmo em C muda drasticamente para melhor. Nós removemos o passo de separação por grupos e o embaralhamento final.O mecanismo passa a ser um fluxo contínuo e direto de extração de entropia:Unificação Precoce: O usuário escolhe as categorias apenas para definir o tamanho final do alfabeto (\(N\)). Se ele marcou tudo, \(N=95\). Se marcou só minúsculas, \(N=26\).Consumo Linear da Entropia: Para cada uma das \(x\) posições da senha, o código consome os bytes da sua entropia forte, aplica a rejeição de módulo (para manter a distribuição uniforme) e cospe o caractere correspondente.Vantagens dessa abordagem focada em Tamanho:Entropia Máxima por Caractere: Ao não forçar um caractere especial ou maiúscula em uma posição específica, você não dá nenhuma pista ao atacante. Cada caractere tem exatamente a mesma probabilidade de ser qualquer coisa dentro do alfabeto eleito.Simplificação do Código: O algoritmo fica muito mais limpo e rápido, eliminando a necessidade de funções de Shuffle (embaralhamento), pois a própria geração já nasce perfeitamente embaralhada pela força da entropia.
